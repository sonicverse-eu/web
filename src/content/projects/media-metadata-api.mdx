Media Metadata API provides a single, consistent interface for working with audio metadata across all the formats your workflows encounter.

## What it includes

- Read and write ID3v2, Vorbis Comment, MP4/iTunes, and FLAC metadata
- Normalise fields across formats into a unified schema
- Podcast-specific fields: GUID, episode type, season, chapter marks
- Batch processing endpoint for large media libraries
- Lightweight self-hostable Node service

## Why it exists

Every audio format stores metadata differently. Teams end up writing one-off parsers for each format they touch. This project removes that friction with a single, well-documented API that handles the edge cases so you don't have to.

## Good fit for

- Podcast publishing platforms
- Radio archive systems
- Media asset management teams
- Developers building audio pipelines
